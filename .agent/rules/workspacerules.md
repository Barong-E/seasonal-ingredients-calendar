## 프로젝트 맞춤 설정 및 자동화 (Customizations & Automation)
- **[중요] 빌드 폴더 수정 절대 금지**: `docs/` 폴더는 Vite 빌드 결과물이 저장되는 폴더임. 이 폴더 안의 파일을 수정하면 빌드 시 초기화되거나 안드로이드 앱과 동기화되지 않으므로 절대 직접 수정하지 말 것.
- **수정 위치 준수**: 반드시 루트 디렉토리에 있는 원본 파일(`index.html`, `setting.html` 등)이나 `assets/` 폴더 내의 파일만 수정할 것.
- **작업 후 자동 반영 (필수)**: 코드를 수정한 후에는 별도 안내 없이 에이전트가 직접 터미널 도구를 사용해 다음 두 가지 작업을 **반드시 순서대로 모두 실행**할 것. 하나라도 누락되어서는 안 됨.
  1) **안드로이드 동기화**: `npm run build && npx cap sync`
  2) **Git 원격지 푸시**: `git add . && git commit -m "[자동] 작업 내용 요약" && git push`

## 데이터와 코드 스타일 (Data & Code Style)
- **데이터 수정/추가 규칙**: 데이터를 수정하거나 추가할 때는 항상 `DATA_GUIDE.md` 문서를 먼저 읽고 형식을 똑같이 맞춰라.
- **HTML/CSS 디자인 규칙**: HTML의 디자인을 바꿀 때는 외부 라이브러리(Tailwind 등)를 쓰지 말고, 항상 기존에 있는 `assets/css` 안의 스타일을 활용해라.

## 플레이 콘솔 배포 빌드 규칙 (App Release & Build Guide)
- **배포 안내 필수 요소**: 플레이 콘솔 업로드용 배포 빌드(AAB 빌드) 요청을 완료했을 때는, 사용자에게 결과 보고(또는 walkthrough.md) 시 다음 세 가지 사항을 반드시 포함하여 알려주어야 한다:
  1) **배포 파일의 정확한 파일 경로(file:// 링크 포함)와 용량 크기**
  2) **구글 플레이 콘솔에 입력할 출시명(Release Name) 및 XML 형식의 출시 노트(Release Notes)**
  3) **어떻게 앱 업데이트를 진행하는지 상세한 단계별 업로드 가이드**
